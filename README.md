## 2. 量化策略主流方向与AI应用

### 2.1 策略分类概览

```
量化策略
├── Alpha策略（选股/择时）
│   ├── 多因子模型（基本面因子、技术因子、另类数据因子）
│   ├── 统计套利（配对交易、指数套利）
│   └── 事件驱动（并购、财报、分红）
├── CTA/趋势跟踪
│   ├── 时间序列动量
│   ├── 横截面动量
│   └── 突破策略
├── 高频交易 (HFT)
│   ├── 做市策略
│   ├── 套利（跨市场、跨品种）
│   └── 订单流预测
├── 市场中性
│   ├── 股票多空
│   └── 可转债套利
└── 宏观量化
    ├── 风险平价
    └── 因子择时
```

### 2.2 AI/ML在量化中的深度应用

#### 当前主流技术栈（2024-2026）

**因子挖掘与Alpha预测：**

| 技术 | 应用场景 | 成熟度 | 代表实践 |
|------|----------|--------|----------|
| XGBoost/LightGBM | 多因子截面选股 | ⭐⭐⭐⭐⭐ 非常成熟 | 几乎所有量化私募的baseline模型 |
| 深度神经网络 (DNN) | 非线性因子组合 | ⭐⭐⭐⭐ | 替代线性回归，处理高维因子 |
| LSTM/GRU | 时间序列预测、择时 | ⭐⭐⭐⭐ | 价格序列建模、波动率预测 |
| Transformer/Attention | 多资产关联、长序列依赖 | ⭐⭐⭐⭐ | 替代LSTM，处理更长序列 [待验证具体应用] |
| 图神经网络 (GNN) | 产业链关联、供应链传导 | ⭐⭐⭐ | 知识图谱+股票关联预测 |
| 强化学习 (RL) | 交易执行、动态仓位管理 | ⭐⭐⭐ | 最优执行算法替代TWAP/VWAP |

**另类数据处理（NLP + CV）：**

| 数据源 | 技术 | 应用 |
|--------|------|------|
| 新闻/公告文本 | BERT/GPT微调、情感分析 | 事件驱动策略、舆情因子 |
| 财报电话会录音 | NLP转写+情感/语调分析 | 管理层信心因子 |
| 卫星图像 | CNN图像识别 | 零售客流、原油库存、农田产量 |
| 供应链数据 | GNN/知识图谱 | 产业链风险传导 |
| 信用卡/消费数据 | 聚合统计建模 | 消费趋势预测、个股营收预估 |
| 社交媒体 | NLP情感分析 | Reddit/Twitter舆情因子（争议较大） |

**大语言模型 (LLM) 在量化中的新兴应用（2024-2026 热点）：**

| 应用方向 | 说明 | 成熟度 |
|----------|------|--------|
| 策略代码生成 | LLM辅助编写回测代码 | ⭐⭐⭐⭐ |
| 财报/研报解读 | 批量解析财报关键信息，生成结构化因子 | ⭐⭐⭐⭐ |
| 风险报告自动化 | 市场异动时自动生成归因分析报告 | ⭐⭐⭐ |
| 知识图谱构建 | 从文本中抽取实体关系构建产业链图谱 | ⭐⭐⭐ |
| 市场预测 | 直接让LLM预测股价方向（争议大，多数研究认为信息优势有限） | ⭐⭐ |

### 2.3 技术基础设施趋势

- **GPU集群训练**：头部量化私募自建千卡级GPU集群，用于深度学习模型训练。幻方量化甚至孵化了DeepSeek大模型公司。
- **低延迟执行**：C++ + FPGA，追求纳秒级延迟（高频领域）。非高频领域延迟要求降到微秒-毫秒级。
- **云原生**：Kubernetes + Docker，弹性扩缩容进行回测和参数优化。
- **数据湖/数据仓库**：PB级另类数据存储，实时数据pipeline（Kafka/Flink）。
- **MLOps**：模型版本管理、自动重训练、AB测试框架。Qlib在这方面的开源方案值得关注。

### 3.4 行业趋势观察

1. **策略拥挤**：传统多因子alpha衰减加速，迫使机构向更复杂模型（深度学习、另类数据）转型。
2. **高频受限**：A股T+1制度和监管收紧（量化交易报备、申报费提高）压缩了高频空间。
3. **AI军备竞赛**：GPU集群规模成为核心竞争力之一，「有多少卡做多大的事」。
4. **人才战争**：顶级量化私募给应届PhD开出200万+年薪已是公开秘密 [待验证当前具体数字]。
5. **合规压力**：2024-2025年证监会加强对量化交易的监管，包括程序化交易报告制度、异常交易监控等。
6. **多策略/多资产**：头部机构从单一股票量化向股票+期货+期权+债券多资产扩展。

## 4. 开源 vs 商业工具对比

### 4.1 核心差异

| 维度 | 开源工具 | 商业/云端平台 | 自研(头部私募) |
|------|----------|---------------|----------------|
| **成本** | 免费（但需要自己维护） | 数据费+平台费（数千-数万/年） | 极高（人力+硬件） |
| **定制性** | 高，可随意修改源码 | 低，黑盒 | 最高 |
| **上手难度** | 中-高 | 低-中 | 极高 |
| **数据质量** | 自行解决 | 平台提供（通常较好） | 自建（最好） |
| **实盘能力** | vnpy较好，其他一般 | 对接券商成熟 | 最成熟 |
| **适合人群** | 学习、研究、个人交易 | 个人-中小机构 | 机构 |

### 4.2 不同场景的推荐路径

```
初学者/学习阶段:
  聚宽/米筐 (在线平台) → Backtrader/Zipline (本地回测)
  → vnpy (了解实盘)

个人量化交易:
  研究: Qlib + 聚宽数据
  回测: Backtrader 或 QuantConnect
  实盘: vnpy (中国市场) / Freqtrade (加密货币)

小团队/私募起步:
  云端: RiceQuant机构版 或 BigQuant
  本地: 掘金量化SDK + 自研因子系统
  执行: vnpy或掘金实盘模块

机构级:
  全部自研，开源仅作参考
```

## 5. 工具选型建议

### 如果现在开始做量化，推荐的技术路径：

**第一层：学习与验证（0-3个月）**
- 平台：聚宽 (JoinQuant) — 最快上手，数据齐全
- 框架：Backtrader — 理解回测引擎原理
- 因子分析：Alphalens — 标准化因子评估

**第二层：策略开发（3-6个月）**
- AI研究：Microsoft Qlib — 完整的ML量化研究流程
- 回测：自研或QuantConnect
- 数据：tushare/akshare (开源数据) + 聚宽数据API

**第三层：实盘部署（6个月+）**
- 中国市场：vnpy (最成熟的开源实盘方案)
- 加密货币：Freqtrade
- 执行系统：逐步向C++迁移（如需低延迟）

**关键认知：**
- 开源工具适合学习和初步策略验证，但不是alpha来源
- 赚钱的策略都在自研框架上运行
- AI/ML是工具而非魔法 — 数据质量和特征工程比模型架构更重要
- 策略拥挤是真实存在的 — 简单的多因子/均线策略在A股已很难稳定盈利
